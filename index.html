操作系统chapter1笔记

王道考研计算机chapter1

计算机系统自下而上可以分为四个部分

  1. 硬件
  2. 操作系统
  3. 应用程序
  4. 用户

操作系统系统管理各种计算机硬件，为应用程序提供基础，充当用户和硬件之间的中间件。

操作系统的特征

操作系统系统属于一种特殊的系统软件。

  1. 并发:并发是指两个或者多个事件在同一时间间隔内发生，而操作系统的并发性是指计算机系统中同时存在多个运行程序。
  2. 共享:是指操作系统中资源可供内存中多个并发执行的进程使用，共享分为两种:互斥共享和同时访问。
  3. 虚拟:虚拟是把一个物理上的实体变化为诺干逻辑上的对应物，物理实体是现实的，而后者是虚拟的，是用户感觉上的事物，用于实现虚拟的技术，称为虚拟技术，比如虚拟处理器和虚拟内存等。
  4. 异步:多道程序环境允许多个程序并发执行，但是由于资源有限，所以进程的执行并不是一贯到底的，以不可预知的速度向前推荐。

并发和共享是操作系统两个最基本的特征，两者之间相互为存在条件:

  1. 资源共享是以程序的并发为条件的，如果系统不允许程序并发执行，就不会存在资源共享问题。
  2. 如果系统不能对资源共享施加有效管理，就会影响到程序的并发执行，甚至根本无法并发执行。

操作系统的目标和功能

  1. 处理机管理:处理机的分配和运行都是以进程或者线程为基本单位，所以对处理器的管理可以看成对进程的管理，进程的管理涉及以下内容
    1. 进程创建
    2. 进程撤销
    3. 进程控制
    4. 进程同步
    5. 进程通信
    6. 死锁处理
    7. 处理机调度
  2. 存储器管理
    1. 内存分配与回收
    2. 地址映射
    3. 内存保护
    4. 内存共享和内存扩充等
  3. 设备管理
  4. 文件管理
    1. 文件存储空间管理
    2. 目录管理
    3. 文件读写管理和保护等

操作系统作为用户与计算机硬件系统之间的接口，有两类接口:

  1. 命令接口
    1. 联机命令接口:适用于分时或者实时系统。
    2. 脱机命令接口:适用于批处理系统。
  2. 程序接口

操作系统的发展与分类

  1. 手工操作阶段(没有操作系统)
  2. 批处理阶段(操作系统出现)
    1. 单道批处理系统:系统对作业的处理是成批进行的，但是内存中始终保持一道作业，单道批处理系统是在解决人机矛盾以及CPU和I/O设备速率不匹配的矛盾中形成的，特征如下
      1. 自动性
      2. 顺序性
      3. 单道性
    2. 多道批处理系统:多道程序设计技术允许多个程序同时进入到内存中并允许它们在CPU中交替运行，这些程序共享系统中的各种硬软件资源，特点如下
      1. 多道
      2. 宏观上并行
      3. 微观上串行
  3. 分时操作系统
    1. 同时性(多路性)
    2. 交互性
    3. 独立性
    4. 及时性
  4. 实时操作系统
    1. 硬实时操作系统
    2. 软实时操作系统
  5. 网络操作系统
  6. 分布式计算机系统
  7. 个人系统

多道程序设计技术的解决如下问题:

  1. 如何分配处理器
  2. 多道程序的内存分配问题
  3. I/O设备如何分配
  4. 如何组织和存放大量的程序和数据，以方便用户使用并保证其安全性与一致性。

操作系统的运行环境

  • 用户态
  • 内核态

内核态的四个主要功能

  1. 时钟管理
  2. 中断机制
  3. 原语
    1. 处于操作系统的最底层，是最接近硬件的部分。
    2. 这些程序的运行具有原子性，其操作只能一气呵成。
    3. 这些时间的运行时间都比较短，而且调用频繁。
  4. 系统控制的数据结构以及处理
    1. 进程管理
      1. 进程状态管理
      2. 进程调度和分派
      3. 创建和撤销进程控制块
    2. 存储器管理
    3. 设备管理

名词解释

  1. 互斥:当一个进程使用资源的时候，其他进程想要使用这个资源，必须要等之前的进程使用结束，这种共享资源的方式被称为互斥。
  2. 临界资源:在一段时间内只允许一个进程访问的资源被称为临界资源或独占资源。
  3. 时分复用技术:比如处理器的分时共享。
  4. 空分复用技术:虚拟存储器。

其中张原嘉答主的很不错，一开始我的路子走错了，我直接去看了tidb的源码，模仿他们的parser，这一步其实还好，编写sql语法规则，关键在于，添加了很多无用功在里面，然后一些基础的功能没有去实现，所以中间浪费了好多时间，不过后期也晓得如何去抄去模仿项目的代码。

最近一些总结

学习失误

最近学习变得很忙碌，但是又感觉什么没学到，经过复盘，我发现了最近学习上的一些错误，
首先就是记录了大量的笔记，但是笔记又没有去看，也就是说花了很多时间做了无用功，
并不是说做笔记的行为不好，而是花大量时间在做笔记上，优点本末倒置了，
计算机是一门相对动手能力较强的学科，所以只需要记下重点内容就行。

如何改进

吸收新知识，需要一段时间来消化新知识，最后输出消化内容，比如说看操作系统中的特性，并发，并行，异步，
第一遍看的时候，可以把三个概念记下来，然后去听老师/公开课讲解，等到了晚上或者下午，在自己结合资料总结。
也符合艾宾浩斯记忆法。

然后利用周末做一个完整的总结，这样总结出来的东西就是自己的，而抄书式笔记的性价比完全太低，因为书本在那里
抄书式笔记只是做了一个搬运工，不如直接看书。

阅读源码技巧

如果实在github(或者类似的)上拉取的开源代码，就从第一个commit开始读，从第一个commit开始读的好处在于

  • 学习别人从零开始建立项目的步骤。
  • 一个项目从零到使用，有很多代码是为了解决bug而写，这些代码对于核心逻辑来说是一种干扰。

针对大型项目代码较多，流程算法比较复杂，就需要遵守如下几个原则：

  • 先设计在代码，先了解设计和基本概念再下手写代码，比如MySQL的代码，先对它的架构设计有初步了解，有sql和存储，支持多存储引擎，理解线程模型和数据库基本概念。
  • 由粗到细，大型项目一般是由多个子项目组成，每个子项目完成自己独立的功能，一个子项目又由多个模块组成，每个模块会有多个类，第一阶段只需要了解每个模块和类的大概作用就行。
  • 由外到内，看实现的对外接口，从这些接口作为切入点，把整个开发流程搞透，并且晓得关键算法在哪实现。
  • 熟悉套路。
