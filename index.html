触发器介绍

触发器是一条语句，当对数据库中数据进行操作的时候，触发器会自动被系统执行，因为触发器是作为强制运行的特定动作的程序，所以又被称为DML触发器，想要使用触发器必须满足两个要求:

  1. 指明什么条件下执行触发器，它被分解维一个引起触发器被检测的事件和一个触发器执行必须满足的条件。
  2. 指令触发器执行的动作(update/insert/delete等等)

触发器可以被设置为有效或者无效，默认情况下是有效的，但是可以通过alter tiger tiger_name disable来修改，使触发器变为无效。

关于我学习计算机的方式

之前写过类似的，但是属于缝合多个答案，马上期末考试了，这篇文章还是11月创建的，换个角度来看，我的拖延症还是蛮严重的。

说回正题，之前每天看起来很忙，实际上每天学习时间长度在5-7小时，其中还有点迷迷糊糊，属于低效学习。

说明想要有高效的学习必须要保证睡眠充足，所以每天中午最好能睡上一小会。

接着是关于学习任务优先级的问题，这个其实可以当成是进程调度问题，之前属于先来先服务方式，每次都把最困难或者耗时最长的放在首位，这样的话，一天除去上课时间，剩下的时间都在写这个，如果能完成还好，没有完成的话，则会浪费很多时间和精力在上面，所以这种性价比不高。

个人觉得，应该先将容易理解的或者相对来说比较熟悉的，放在前面，这样处理起来较快。

就拿最近这段时间举例，看起来每天都在忙，但是只有自己才晓得有多低效，因为一天中的大部分重要时间都花费在大作业上面，软工和python两个大作业，这样的话，就和之前情况一样，如果当天能够解决，就还好，如果当天没有解决，那大部分精力和时间已经用出去了，但是收获不大。

而且也没有心情在学其他课程，所以做了一个调整，就是将要看的资料，比如PPT，论文，视频等等，放在首位，首先这样的话，能够保障有所收获，至少是可以看到头的，但是写项目代码属于周期较长，虽然可以分解成一个个小目标，但是在写代码的过程中，很多不可预料性太多，容易耽误时间和精力。

所以这里总结一下:

  1. 身体最重要，任何学习方式，如果身体不佳，再怎么高效的学习方式都没有用。
  2. 调整任务优先级，把较为简单的调到前面。
  3. 需要复习旧知识和预习新知识。

操作系统chapter2笔记

进程的概念和特征

在多道程序环境下，多个程序并发执行会失去封闭性，并且具有间断性和不可再现性的特征，所以引入了进程的概念，
更好的描述和控制程序的并发执行，实现了操作系统基本的两个特性，并发性和共享性。

为了能够让参与并发的程序能够独立的运行，必须配置一个专门的数据结构，这种数据结构被称为PCB。

进程经典的定义

  1. 进程是程序的一次性执行过程。
  2. 进程是一个程序以及数据在处理机上顺序执行的时候所发生的活动。
  3. 进程是具有独立功能的程序在一个数据集合上运行的过程，它是系统进行资源分配和调度的一个独立单位。

进程特征

  1. 动态性
  2. 并发性
  3. 独立性
  4. 异步性
  5. 结构性

进程的状态与转换

进程的基本三种状态如下

  1. 运行态
  2. 就绪态
  3. 阻塞态

进程的五种状态如下

  1. 运行态
  2. 就绪态
  3. 阻塞态
  4. 创建态
  5. 结束态

进程控制

进程控制的主要功能是对系统中的所有进程实施有效的管理，它具有创建新进程，撤销现有进程，实现进程状态转换等功能，
在操作系统中，一般把进程控制用的程序称为原语，原语的特点是执行期间不允许中断，它是一个不可分割的基本单位，以下是进程控制的操作

  1. 进程的创建
  2. 进程的终止
    1. 正常结束
    2. 异常结束
    3. 外界干预
  3. 进程的阻塞和唤醒
  4. 进程切换

进程的组织

进程是一个独立的运行单位，也是操作系统进行资源分配和调度的基本单位，它是由以下三个部分组成

  1. 进程控制块(PCB)
    1. 进程描述信息
    2. 进程控制
    3. 管理信息
    4. 资源分配清单
    5. 处理机相关信息
  2. 程序段
  3. 数据段

进程的通信

进程通信是指进程之间的信息交换，PV操作是低级通信方式，高级通信方式是指较高的效率传输大量数据的通信方式，高级通信方式主要有以下三种

  1. 共享存储
    1. 低级共享存储:是基于数据结构的共享。
    2. 高级共享存储:是基于存储区的共享。
  2. 消息传递
    1. 直接通信方式
    2. 间接通信方式
  3. 管道通信
    1. 互斥
    2. 同步
    3. 确定对方的存在

线程概念和多线程模型

线程是一个基本的CPU执行单元，也是程序执行流的最小单元，由线程ID，程序计数器，寄存器集合和堆栈组成，线程只是进程中一个实体，是被系统独立调度和分派的基本单位，
线程不拥有系统资源，但是它可与同属一个进程的其他线程共享进程所拥有的全部资源。

线程的实现

线程的实现可以分为两类

  1. 用户级线程
  2. 内核级线程

线程和进程的比较

在同一个进程中，线程的切换不会引起进程切换，而在不同的进程中进行线程切换，就会引起进程切换。

多线程模型

  • 多对一模型
    • 优点:线程管理是在用户空间进行的，所以效率比较高。
    • 缺点：一个线程在使用内核服务的时候被阻塞，整个进程都会被阻塞，多个线程不能并行运行在多处理机上。
  • 一对一模型
    • 优点: 当一个线程被阻塞后，运行另一个线程继续执行，所以并发能力比较强。
    • 缺点: 一个线程在使用内核服务的时候被阻塞，整个进程都会被阻塞，多个线程不能并行运行在多处理机上。
  • 多对多模型
    • 特点：多对多模型在一对一和多对一模型中折中，取两者优点，克服了其他两种的缺点。

处理机调度

调度的基本概念

在多道程序系统中，进程的数量要比处理机的数量多，所以进程就会发生争用处理机的情况，处理机调度是对处理机进行分配，即从就绪队列中按照某种算法选中一个进程并将处理机分配给它运行，来完成进程的并发执行。

调度分类

  1. 高级调度(作业调度)
  2. 中级调度(内存调度)
  3. 低级调度(进程调度)

现代操作系统中，不能进行进程的调度和切换有以下几种情况:

  1. 在处理中断的过程中。
  2. 进程在操作系统内核程序临界区中。
  3. 其他需要完全屏蔽中断的原子操作过程。

进程调度的方式

  1. 非剥夺调度方式：不管多重要的进程进入就绪队列中，都要让当前的进程运行结束，然后再到这个进程，适用于大多数的批处理系统，这种方式的优点在于实现简单，系统开销小。
  2. 剥夺调度方式：当一个进程正在处理机上的时候，有更要的进程要使用处理机，则应该让当前进程停下来，让更重要的进程使用。

调度的基本准则

  1. cpu利用率
  2. 系统吞吐量
  3. 周转时间
  4. 等待时间
  5. 响应时间

典型的调度算法

  1. 先来先服务(FCFS)调度算法:既可以用于作业调度，也可以用于进程调度，FCFS属于不可剥夺算法。
    1. 算法简单，但是效率低。
    2. 对长作业有效，但是面对短作业效率就低。
    3. 有利于CPU繁忙型作业，不利于I/O繁忙型作业。
  2. 短作业优先算法(SJF)::既可以用于作业调度，也可以用于进程调度。
    1. SJF对长作业不利，因为SJF算法中长作业的周转时间会增加，假设有一个长作业进入系统的就绪队列中，因为调度程序总是优先调度短作业，就会导致长作业长期不被调用，从而发生饥饿现象。
    2. SJF算法完全没有作业的紧迫程度，所以不能保证紧迫性作业会及时处理。
    3. 由于作业的长度是根据用户提供的估计运算时间而定，而用户有可能会有意无意的缩短作业的估计运行时间，到底SJF算法不一定能做到短作业优先调度。
  3. 优先级调度算法：既可以用于作业调度，也可以用于进程调度
  4. 高响应比优先调度算法:因为每次从就绪队列中选择优先级最高的进程能否抢占正在执行的进程，所以分为两种算法：
    1. 非剥夺优先级调度算法
    2. 剥夺式优先级调度算法
  5. 时间片轮转调度算法:适用于分时系统
  6. 多级反馈队列调度算法

操作系统chapter1笔记

王道考研计算机chapter1

计算机系统自下而上可以分为四个部分

  1. 硬件
  2. 操作系统
  3. 应用程序
  4. 用户

操作系统系统管理各种计算机硬件，为应用程序提供基础，充当用户和硬件之间的中间件。

操作系统的特征

操作系统系统属于一种特殊的系统软件。

  1. 并发:并发是指两个或者多个事件在同一时间间隔内发生，而操作系统的并发性是指计算机系统中同时存在多个运行程序。
  2. 共享:是指操作系统中资源可供内存中多个并发执行的进程使用，共享分为两种:互斥共享和同时访问。
  3. 虚拟:虚拟是把一个物理上的实体变化为诺干逻辑上的对应物，物理实体是现实的，而后者是虚拟的，是用户感觉上的事物，用于实现虚拟的技术，称为虚拟技术，比如虚拟处理器和虚拟内存等。
  4. 异步:多道程序环境允许多个程序并发执行，但是由于资源有限，所以进程的执行并不是一贯到底的，以不可预知的速度向前推荐。

并发和共享是操作系统两个最基本的特征，两者之间相互为存在条件:

  1. 资源共享是以程序的并发为条件的，如果系统不允许程序并发执行，就不会存在资源共享问题。
  2. 如果系统不能对资源共享施加有效管理，就会影响到程序的并发执行，甚至根本无法并发执行。

操作系统的目标和功能

  1. 处理机管理:处理机的分配和运行都是以进程或者线程为基本单位，所以对处理器的管理可以看成对进程的管理，进程的管理涉及以下内容
    1. 进程创建
    2. 进程撤销
    3. 进程控制
    4. 进程同步
    5. 进程通信
    6. 死锁处理
    7. 处理机调度
  2. 存储器管理
    1. 内存分配与回收
    2. 地址映射
    3. 内存保护
    4. 内存共享和内存扩充等
  3. 设备管理
  4. 文件管理
    1. 文件存储空间管理
    2. 目录管理
    3. 文件读写管理和保护等

操作系统作为用户与计算机硬件系统之间的接口，有两类接口:

  1. 命令接口
    1. 联机命令接口:适用于分时或者实时系统。
    2. 脱机命令接口:适用于批处理系统。
  2. 程序接口

操作系统的发展与分类

  1. 手工操作阶段(没有操作系统)
  2. 批处理阶段(操作系统出现)
    1. 单道批处理系统:系统对作业的处理是成批进行的，但是内存中始终保持一道作业，单道批处理系统是在解决人机矛盾以及CPU和I/O设备速率不匹配的矛盾中形成的，特征如下
      1. 自动性
      2. 顺序性
      3. 单道性
    2. 多道批处理系统:多道程序设计技术允许多个程序同时进入到内存中并允许它们在CPU中交替运行，这些程序共享系统中的各种硬软件资源，特点如下
      1. 多道
      2. 宏观上并行
      3. 微观上串行
  3. 分时操作系统
    1. 同时性(多路性)
    2. 交互性
    3. 独立性
    4. 及时性
  4. 实时操作系统
    1. 硬实时操作系统
    2. 软实时操作系统
  5. 网络操作系统
  6. 分布式计算机系统
  7. 个人系统

多道程序设计技术的解决如下问题:

  1. 如何分配处理器
  2. 多道程序的内存分配问题
  3. I/O设备如何分配
  4. 如何组织和存放大量的程序和数据，以方便用户使用并保证其安全性与一致性。

操作系统的运行环境

  1. 时钟管理
  2. 中断机制
  3. 原语
    1. 处于操作系统的最底层，是最接近硬件的部分。
    2. 这些程序的运行具有原子性，其操作只能一气呵成。
    3. 这些时间的运行时间都比较短，而且调用频繁。
  4. 系统控制的数据结构以及处理
    1. 进程管理
      1. 进程状态管理
      2. 进程调度和分派
      3. 创建和撤销进程控制块
    2. 存储器管理
    3. 设备管理

  2. 微内核

名词解释

  1. 互斥:当一个进程使用资源的时候，其他进程想要使用这个资源，必须要等之前的进程使用结束，这种共享资源的方式被称为互斥。
  2. 临界资源:在一段时间内只允许一个进程访问的资源被称为临界资源或独占资源。
  3. 时分复用技术:比如处理器的分时共享。
  4. 空分复用技术:虚拟存储器。
