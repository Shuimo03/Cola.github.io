最近一些总结

学习失误

最近学习变得很忙碌，但是又感觉什么没学到，经过复盘，我发现了最近学习上的一些错误，
首先就是记录了大量的笔记，但是笔记又没有去看，也就是说花了很多时间做了无用功，
并不是说做笔记的行为不好，而是花大量时间在做笔记上，优点本末倒置了，
计算机是一门相对动手能力较强的学科，所以只需要记下重点内容就行。

如何改进

吸收新知识，需要一段时间来消化新知识，最后输出消化内容，比如说看操作系统中的特性，并发，并行，异步，
第一遍看的时候，可以把三个概念记下来，然后去听老师/公开课讲解，等到了晚上或者下午，在自己结合资料总结。
也符合艾宾浩斯记忆法。

然后利用周末做一个完整的总结，这样总结出来的东西就是自己的，而抄书式笔记的性价比完全太低，因为书本在那里
抄书式笔记只是做了一个搬运工，不如直接看书。

阅读源码技巧

如果实在github(或者类似的)上拉取的开源代码，就从第一个commit开始读，从第一个commit开始读的好处在于

  • 学习别人从零开始建立项目的步骤。
  • 一个项目从零到使用，有很多代码是为了解决bug而写，这些代码对于核心逻辑来说是一种干扰。

针对大型项目代码较多，流程算法比较复杂，就需要遵守如下几个原则：

  • 先设计在代码，先了解设计和基本概念再下手写代码，比如MySQL的代码，先对它的架构设计有初步了解，有sql和存储，支持多存储引擎，理解线程模型和数据库基本概念。
  • 由粗到细，大型项目一般是由多个子项目组成，每个子项目完成自己独立的功能，一个子项目又由多个模块组成，每个模块会有多个类，第一阶段只需要了解每个模块和类的大概作用就行。
  • 由外到内，看实现的对外接口，从这些接口作为切入点，把整个开发流程搞透，并且晓得关键算法在哪实现。
  • 熟悉套路。

计算机书籍阅读方法

之前在金老师的live中听过关于如何阅读计算机书籍的方法，但是之前做的笔记不见了，所以这次重新写过。
金老师在live中有一张PPT，上面将看书一共分为五个阶段，分别如下

  • 预热阶段
  • 第一遍
  • 第二遍
  • 第三遍
  • 第四遍以及更多

预热阶段是先构造整体知识结构，同时确定阅读顺序和具体学习路线，这个方法的好处在于，你可以看其中的重点部分，不用全部啃完一本大头书，
这个阶段主要目标是确定要学那些章节，对一本书或者这本书的技术有一个大概认知。

第一遍仔细阅读第一章，快速的浏览一章的开头和结尾，跳过中间，因为中间部分会在结尾处，进行一个总结，所以中间可以选择性跳读，等到了第二遍在进行补充，
第二遍的时候，主要是对第一遍跳读的部分或者不懂的地方进行细读，同时也挑出最感兴趣的部分进行细读，如果不同的话，可以先留着。
到了第三遍就是对第一第二遍整体的复盘了，把没有看懂的弄懂。
第四遍以及更多遍则是通读全书，完成书上练习题，进行巩固。

阅读细节

如果是编程书的话，要分为两种情况，如果是之前没有接触过编程的话，还是要动手写书上的代码，而有编程经验的话，则需要跟上作者的思路，有课后习题，则要通过
课后习题来完成语法学习，在第一遍过程中，只需要看完预热阶段安排的章节即可，不需要全部看完，在第二遍中，选择性的敲书上代码，因为这个时候，就晓得书上代码讲的是什么了
在理解代码的情况下，可以有助于记忆语法。到了第三遍，语法和这门编程语言的概念已经掌握了，所以就可以动手做一些项目了。

结语

在学习中实践，在实践中学习，看书的过程中，遇到不懂的点可以记下来，或者找类似的书去看，不要做抄书式笔记，只需要把不懂的点和重点记录下来即可。

实现论文中的算法方法

准备工作

如果不是为了纯碎的学习目的去实现论文中的算法的话，就不一定非要实现，可以先花一些时间去找找看网上是不是有现有的代码。

正确的选择实现语言和库

进行这一阶段的前提是，已经看懂论文了，对里面提出来的算法和概念有一个清晰的认识，而且必须有一个可以运行的原型程序，来保证算法能够在预想的输入数据下工作，
实现算法的语言可以选择是自己熟练的，也可以是成熟的编程语言和库。

实现

选择正确的精度，比如使用double替代float，虽然double占用的内存多，但是保证了精度，另外系统之间有差别，比如32bit系统和64bit系统之间的不同处。
如果可以的话，自己可以创建一个(封装好float或者double 32bit或64bit)类型，然后在代码中使用封装的数据类型，Java可以使用class来实现，C/C++可以使用define来实现
。

为代码做好注释

实现一个复杂的技术论文时，最好是做好文档，应该对自己的文件、类、方法进行过注释说明，一个是方便其他人可以看懂，另一方面是为了防止以后
自己看代码的时候，避免看不懂实现细节。
在代码中添加引用，对于要实现的论文中每一条方程。

其他

实现算法完之后，需要进行调试和测试，来对算法进行检查，同时对比论文中的算法，或者原产品进行对比，这一步就像是数学题验算一样。
最后和论文作者进行交流，如果已经完成了以上步骤，这个时候和论文作者交流，可以晓得那些地方有问题，那些地方需要改进。

CS106L课程配置

环境配置

因为CS106B和CS106L都是用qt作为ide，所以这里讲下如何配置CS106L的开发环境，如果先去下载qt。

下载完成后，选择安装mingw的选项，不要安装微软的，因为运行起来会报错，具体原因没有搞明白，可能是不兼容或者什么的。

下载blank_project

去课程主页下载blank_project，然后解压，打开里面的Blank.pro，选择minGW，然后配置就行了，等配置完，按下绿色的三角形，如果运行成功会弹出一个窗口。

自学编程方法

自学编程方法

掌握级别

  • 知道：平时基本上用不到，知道这个语言是干什么用的，这里以我自己举例，比如visual Basic。
  • 了解：懂这个编程语言的语法，能够看懂别人写的，自己也可以模仿的写一些，比如python。
  • 会用：主要使用，但是很多细节和特性不懂，需要加强提升的，比如Java。
  • 熟练：在会用的基础上提升，能够流利的使用常用api和框架。
  • 精通：了解这门语言的优缺点，以及如何实现的，能够看懂源码。

需要掌握的语言

  • 底层语言
    • C(这里把C放到底层语言，是因为学会了可以对计算机的数据存储和指令执行有更深的了解。)
    • 汇编
  • 面向对象语言
    • C++
    • Java/C sharp
  • 动态语言
    • JavaScript
    • python
    • typescript
  • 函数式编程
    • Lisp
    • s
  • 系统编程语言
    • golang

学习多门语言的目的

学习编程语言之间的目标是解决问题，所以学习一门编译语言有两种方法，一种是工作需求，用来开发产品的主力语言，这类语言需要以精通为目的，另一类是作为辅助工具来使用，为了提高工作效率，比如python和shell，这一类达到熟练最好。

学习方法

  • 清楚自己的目标：每一门语言都是入门简单，精通难，所以一定要搞清楚自己的目标是什么，很多时候了解一门新的知识只要了解它的20%就可以解决大部分问题，如果遇到需要深入了解的地方，可以聚在一起去查缺补漏，不要为了深入而深入。
  • 广度：在之前提到过的编程语言中，至少各掌握一门，不一定要一样的，一样类型的就可以。
  • 项目趋势：学习编程语言，不是抱着书从头看到尾，而是有项目驱使或者需求驱使，所以可以先找一些项目去模仿，这样才能达到掌握这门编程语言。
  • 吸收营养：除了写代码外，还需要看经典的书和优秀的开源项目，通过这些去学习别人的优点，以及编写好代码的方式。
  • 和人交流：虽然是自学，但是不能闭门造车，而是和人沟通交流，这样可以减少很多弯路。
  • 一鼓作气：学习的过程需要连贯性，最好是集中精力在一到两个星期内搞定，因为如果每天只学一点，回头再来学习的时候，已经忘得差不多了。
  • 能学多少算多少，不要钻牛角尖：如果在学习过程中，实在搞不懂可以先跳过，比如Java的容器，里面是关于数据结构的知识，而你如果没有学过数据结构，自然是搞不懂的，所以就需要先跳过，等后面再来解决。

  • 买书
  • 看好的教程(比如 https://www.tutorialspoint.com/cplusplus/index.htm)
  • 官网文档，tutorial
  • 代码托管网站 GitHub 码云
  • 任务引导性学习网站
