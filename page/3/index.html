自学编程方法

自学编程方法

掌握级别

  • 知道：平时基本上用不到，知道这个语言是干什么用的，这里以我自己举例，比如visual Basic。
  • 了解：懂这个编程语言的语法，能够看懂别人写的，自己也可以模仿的写一些，比如python。
  • 会用：主要使用，但是很多细节和特性不懂，需要加强提升的，比如Java。
  • 熟练：在会用的基础上提升，能够流利的使用常用api和框架。
  • 精通：了解这门语言的优缺点，以及如何实现的，能够看懂源码。

需要掌握的语言

  • 底层语言
    • C(这里把C放到底层语言，是因为学会了可以对计算机的数据存储和指令执行有更深的了解。)
    • 汇编
  • 面向对象语言
    • C++
    • Java/C sharp
  • 动态语言
    • JavaScript
    • python
    • typescript
  • 函数式编程
    • Lisp
    • s
  • 系统编程语言
    • golang

学习多门语言的目的

学习编程语言之间的目标是解决问题，所以学习一门编译语言有两种方法，一种是工作需求，用来开发产品的主力语言，这类语言需要以精通为目的，另一类是作为辅助工具来使用，为了提高工作效率，比如python和shell，这一类达到熟练最好。

学习方法

  • 清楚自己的目标：每一门语言都是入门简单，精通难，所以一定要搞清楚自己的目标是什么，很多时候了解一门新的知识只要了解它的20%就可以解决大部分问题，如果遇到需要深入了解的地方，可以聚在一起去查缺补漏，不要为了深入而深入。
  • 广度：在之前提到过的编程语言中，至少各掌握一门，不一定要一样的，一样类型的就可以。
  • 项目趋势：学习编程语言，不是抱着书从头看到尾，而是有项目驱使或者需求驱使，所以可以先找一些项目去模仿，这样才能达到掌握这门编程语言。
  • 吸收营养：除了写代码外，还需要看经典的书和优秀的开源项目，通过这些去学习别人的优点，以及编写好代码的方式。
  • 和人交流：虽然是自学，但是不能闭门造车，而是和人沟通交流，这样可以减少很多弯路。
  • 一鼓作气：学习的过程需要连贯性，最好是集中精力在一到两个星期内搞定，因为如果每天只学一点，回头再来学习的时候，已经忘得差不多了。
  • 能学多少算多少，不要钻牛角尖：如果在学习过程中，实在搞不懂可以先跳过，比如Java的容器，里面是关于数据结构的知识，而你如果没有学过数据结构，自然是搞不懂的，所以就需要先跳过，等后面再来解决。

计算机自学方法(1)

前言

自学计算机最怕的就是走错了弯路，就和打游戏一样，点错了技能点，所以这个时候可以参考国外的计算机培养方案来学习，这里贴上CMU的培养方案，具体的还请自己百度/Google，也可以按照自己感兴趣的学。

https://csd.cmu.edu/academic/undergraduate/bachelors-curriculum-admitted-2018

以下是我自己觉得还不错的学习顺序。

这里首先推荐The Missing Semester of Your CS Education这门课，这是一门MIT的课，主要是教你除了CS课程外，还需要学习那些实用的东西，比如vim/命令行/git，在cmu的课程里面对应的是15-131，一共11节课，涉及范围比较广，但是没有深入，可以选择感兴趣的深入了解下。

第二门课MIT6.001或者cs50，先说下cs50吧，毕竟之前上过一点，但是没有完全看完，这是一门入门课，计算机科学导论，告诉你计算机可以做什么，从二进制到编程语言等等，跟着这门课程可以至少可以了解到一个计算机科学的体系(或者说应该学什么)。

而MIT6.001也是一个计算机基础课程，使用python，因为我还没开始上，所以具体的就不做评价了。

这里的话，给个建议，就是先把基础工具了解一下，然后学习1-2门编程语言，接着去写对应的项目，也算是不错的了。

结语

具体路线因为很长，并且每个人不同，所以说，可以先把公共的部分给看完，后面选择自己感兴趣的就可以了。

设计模式-抽象工厂模式

抽象工厂模式(Abstract-factory)

场景

假设你正在经营一家电器商场，电器商场里面有各种各样的问题，用代码类来表示的话：

  • 一系列相关的产品，比如冰箱，洗衣机，电视等。
  • 系列不同的变体和品牌，比如海尔冰箱，索尼电视等

你需要让每件电器单独生成对象，这样才能让风格保持一致，如果顾客买来发现风格不一样，可能就会不满意，另外也不希望添加新产品或者新的品牌的时候，修改现有代码，电器供应商如果对产品更新速度很快的话，你不会像每次更新都去修改关键代码。

解决方法

首先，抽象工厂模式建议为电器商场中所有的产品都声明接口，比如冰箱，洗衣机，电视等等，接着所有产品都继承这些接口，比如冰箱都实现冰箱接口等等。

接着声明抽象工厂，包括系列中所有产品构造方法的接口，比如创建电视createTv等等，这些方法必须返回抽象产品类型，每个具体工厂类对应一个特定的产品变体，对于系列产品的每个变体，都基于抽象工厂接口创建不同的工厂类，每个工厂类只能返回特定的产品，比如索尼品牌的电器，只能创建索尼电视，索尼冰箱，索尼洗衣机等等。

客户端代码可以通过相应的抽象接口来调用工厂和产品类，不需要改动实际客户端代码，就可以传递给客户端的工厂类，也能够修改客户端代码来接收产品变体，客户端不需要了解所调用工厂的具体类信息。

抽象工厂结构

  1. 抽象产品：为构成系列产品的一组不同但是相关的产品声明接口
  2. 具体产品：是抽象产品的多种不同类型实现，所有变体都必须实现相应的抽象产品。
  3. 抽象工厂：接口声明一组创建各种抽象产品的方法。
  4. 具体工厂：实现抽象工厂的构建方法，每个具体工厂对应特定产品变体，并且仅创建这种产品的而变体。

抽象工厂模式的优缺点

优点

  • 可以确保统一工厂生成的产品相互匹配
  • 可以避免客户端和具体产品代码的耦合
  • 单一职责原则，可以将产品生成代码抽取到同一位置，让代码更加容易维护。
  • 开闭原则，向应用程序中引入新的产品变体时，不需要改变客户端代码

缺点

因为采用该模式需要引入更多的接口和类，所以代码会更加复杂。

抽象工厂的应用场景

如果代码需要和多个不同系列的相关产品交互，但是因为没办法提前获取相关信息，或者处于对未来扩张性的考虑，不希望代码基于产品类进行构造，所以在这种情况下，就可以使用抽象工厂。

抽象工厂提供了一个接口，可以创建每个系列产品的对象，只需要代码通过这个结构创建对象，那么就不会生成和应用程序已经生成的产品类型不一样的产品。

假设有一个基于一组抽象方法的类，并且主要功能还变得不明确，就可以考虑使用抽象工厂模式，在设计良好的程序当中，每个类仅仅复制一件事，如果一个类需要和多种类型产品交互，就可以考虑工厂方法抽取到独立的工厂类或者具备完整功能的抽象工厂类中。

构造器概念

在创建对象的时候被自动调用的特殊方法，Java额外提供了垃圾回收器，对于不再使用的内存资源，垃圾回收器能自动将其释放。

因为构造器调用是编译器的问题，必须让编译器知道应该调用什么方法，所以构造器的名字和类名一致，每个类都一个构造器，如果没有显式构造器的话，编译器会自动生成一个默认的构造方法，没什么任何参数和执行语句。

构造器是一种特别的方法，没有返回值，但是和void不一样，void本身是不会自动返回什么，但是可以选择让它返回别的东西，而构造器就不行。

方法重载

方法重载可以让同一个名字的方法不起冲突，而且方法重载的返回值类型通常是相同的，但是要参数列表不同才可以使用。

当方法重载里面的参数列表接收到的实参小于形参的话，实参就会提升，如果大于方法重载的形参就会转换类型。
